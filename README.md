# AI-Powered Data Quality Engine (Gemini 2.5)

## Overview
This project uses LLM capabilities to automatically detect and resolve data quality issues in datasets.

Instead of traditional rule-based validation, this system leverages Gemini 2.5 to intelligently:
- Identify anomalies
- Detect missing/inconsistent values
- Suggest corrections
- Improve overall data reliability

## Key Features
- LLM-driven data validation
- Automated anomaly detection
- Intelligent data cleaning suggestions

## ⚙️ Architecture
1. Input raw dataset
2. Preprocessing & sampling
3. LLM prompt generation
4. Gemini API call
5. Response parsing
6. Data correction layer

## 🛠️ Tech Stack
- Python
- Gemini 2.5 API
- Pandas / PySpark
- Prompt Engineering

## 📊 Example Use Cases
- Customer data validation
- Duplicate detection
- Missing value handling
- Schema inconsistency checks
